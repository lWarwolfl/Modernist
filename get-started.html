Getting the Modernist Template started

1. Choosing colors

a. Editing css file location(recommended - faster load time)

<link id="theme" rel="stylesheet" href="css/theme/root-color-color.css">

Edit this link in all pages and change the colors to your chosen colors.

b. Editing setting.json file

"color": "color-color",

Open “setting.json” file with any text editor and change the “color” property to your chosen colors. Then save and close the file.

The available hexadecimal colors are:

“f7c025-fc840e”
“ffe53b-ff2525”
“fee140-fa709a”
“00dbde-e600ff”
“8bc6ec-9599e2”
“08aeea-2af598”
“45e690-e6e045”
“f4d03f-16a085”
“fdbb2d-8340ff”
“ff3cac-784ba0”
“2b86c5-ff3cac”
“ff9a8b-ff6a88”

note: These are premade color files and entering other colors and combinations will not work. Although you can make your own color files by copying one of these files and editing the colors. but you should also edit the hero image illustrator file provided in the parchased file and export your own svg with desired colors to To be set as the background.

--hero-image: url('../../image/svg/hero-image-your color-your color.svg');

And then you would have to enter the address in your color file according to above.

2. Configuring customizations

a. Adding classes to body tag inside html(recommended - faster load time)

Header and footer box

<body class="hidden no-box">

Add “no-box” class to remove header and footer boxes to have a more unified page.

Waves

<body class="hidden has-wave">

Add “has-wave” class to enable waves for header and footer boxes.

note: You have to remove “no-box” class for this to work properly.

<body class="hidden has-wave normal"> (more complex waves)
<body class="hidden has-wave simple"> (simple wave)

Then add “normal” class for more complex waves or “simple” class for a simple wave.

Decorative circles

<body class="hidden has-circle">

Add “has-circle” class to enable circles to fill the empty spaces of the page.
note: This will not apply to the mobile version of the template.

<body class="hidden has-circle first"> (first color background)
<body class="hidden has-circle second"> (second color background)
<body class="hidden has-circle gradient"> (both colors as gradient background)

To specify the color of circles use “first” or “second” class to apply the first or second color or use the “gradient” class to apply both as a gradient.

note: You have to add these customization classes to bodies of all pages.

b. Editing setting.json file(easier)

Header and footer box

"box": "yes" (have box - default)
"box": "no" (no box - floating)

Open “setting.json” file with any text editor and change the box property to “no” to remove header and footer boxes to have a more unified page. Then save and close the file.

Waves

This property adds waves for header and footer boxes.

note: “box” property must be set to “yes” for this to work.

"wave": "none" (no waves - default)
"wave": "normal" (more complex waves)
"wave": "simple" (simple wave)

Open “setting.json” file with any text editor and change the “wave” property to “normal” for more complex waves or “simple” for a simple wave. Then save and close the file.

Decorative circles

This property enables circles to fill the empty spaces of the page.
note: This will not apply to the mobile version of the template.

"circle": "none" (no circles - default)
"circle": "first" (first color background)
"circle": "second" (second color background)
"circle": "gradient" (both colors as gradient background)

To specify the color of circles Change “circle” property to “first” or “second” to apply the first or second color or to “gradient” to apply both as a gradient.

3. fix script(optional)

`let setting = {};
//get customization setting from json file
$.getJSON(‘setting.json’, function(data){
let setting = data[‘setting’];
set_setting(setting);
});
function set_setting(setting){
    //set webpage color file
    localStorage.setItem('color',setting['color']);
    $('#theme').replaceWith('<link id="theme" rel="stylesheet" href="css/theme/root-' + localStorage.getItem('color') + '.css">');

    //set box settings
    localStorage.setItem('box',setting['box']);
    if (localStorage.getItem('box') === 'no')
    {
        body.addClass('no-box');
    }

    //set wave settings
    localStorage.setItem('wave',setting['wave']);
    if (localStorage.getItem('wave') !== 'none')
    {
        body.addClass('has-wave');
        body.addClass(localStorage.getItem('wave'));
    }

    //set circle settings
    localStorage.setItem('circle',setting['circle']);
    if (localStorage.getItem('circle') !== 'none')
    {
        body.addClass('has-circle');
        body.addClass(localStorage.getItem('circle'));
    }
}`

If you used method “a” to apply color and customizations you have to delete this segment of code from “js/all/toggle-theme.js” for a faster load time.

4. Use real text and images (the best approach is to use an admin panel and server side language but it is possible in this template to use json files)

note: basic json knowledge is mandatory.

a. Categories

{"category": "string","status": "string","title": "string","description": "string","count": "int","image": "jpeg","icon": "string"}

Enter your category data in the “js/category/card-data.json” file according to this segment of this json code.

note: Check below to understand where each entry is displayed and what is its purpose

  • “category” : Category card - Category page information box(category name)
  • “status” : Category page information box(a short information about category)
  • “title” : Category page hero box(category title)
  • “description” : Category page hero box(a short description for this category)
  • “count” : Category card(all stories inside this category)
  • “image” : Category card - Category page information box - Category page hero box(check below for more information)
  • “icon” : Category card(must be a free fontawesone icon class name)

note: Images for category have four different sizes and are sorted in four different folders.
For adding pictures you have to crop and resize your images according to the specs below and name them exactly the same and inside the designated folders.
then you have to enter only the file name for the “image” property inside the json file.

  • “image/categroy-icon” - small - square - 200*200px - max size 30kb recommended
  • “image/categroy-square” - medium - square - 700*700px - max size 100kb recommended
  • “image/categroy-original” - original - landscape - 800*xxxpx - max size 200kb recommended
  • “image/categroy-high-res” - large - landscape - 1280*xxxxpx - max size 300kb recommended

b. Story

"category": ["string","string","string","string","string","string","string","string","string","string","string","string"],

Enter your category data in the “js/story/card-data.json” file as above.

note: Always keep “All” and “Recent” category.

note: Always keep the categories in “js/story/card-data.json” synced with “js/category/card-data.json” file.

note: These entries are displayed in category page and story page sidebars and you can make stories for each category according to below.

"category 1": [ {"category": "string","time": "int min read - date","title": "string","text": "string","image": "jpeg","writer": {"name": "string","job": "string","image": "jpeg","time": "Posted on\n date","stars": int(1-5)},"story": "story full text"} ],

Enter your story data in the “js/story/card-data.json” file as above.

note: Check below to understand where each entry is displayed and what is its purpose
  • “category 1” : a category from the category array that you want to make an story for it(must be lowercase)
  • “category” : Story card tag(story category indicator)
  • “time” : Story card(Upload time and read time indicator)
  • “title” : Story card - Story page(story title)
  • “text” : Story card(short description of story)
  • “image” : Story card - Story page(check below for more information)
  • “story” : Story page(full story text)
– “writer” : Story card - Story page information card(information about the story writer) – “name” : writer name – “job” : writer job or rank – “image” : writer image(check below for more information) – “time” : upload date – “stars” : writer stars count

note: Images for story have two different sizes and are sorted in two different folders.
For adding pictures you have to crop and resize your images according to the specs below and name them exactly the same and inside the designated folders.
then you have to enter only the file name for the “image” property inside the json file.

  • “image/story” - original - landscape - 800*xxxpx - max size 200kb recommended
  • “image/story-high-res” - big - landscape - 1280*xxxxpx - max size 300kb recommended

note: Images for writers have only one size.
For adding pictures you have to crop and resize your image according to the specs below and put it inside the designated folder.
then you have to enter only the file name for the “image” property inside the json file.
“image/people” - small - square - 200*200px - max size 100kb recommended

c. Team

{"name": "string","job": "string","description": "string","stars": int(1-5),"image": "jpeg"}

Enter your team data in the “js/team/card-data.json” file as above.

note: Check below to understand where each entry is displayed and what is its purpose

  • “name” : team member name
  • “job” : team member job or rank
  • “description” : short description of team member
  • “image” : team member image(check below for more information)
  • “stars” : team member stars count

note: Images for team have only one size.
For adding pictures you have to crop and resize your image according to the specs below and put it inside the designated folder. Then you have to enter only the file name for the “image” property inside the json file.

  • “image/people” - small - square - 200*200px - max size 100kb recommended
